发表于 2017-06-08

这里记录了在使用 Pods 库的时候遇到的各种问题：

搜索不到自己的私有库

执行下面这条命令之后，重新搜索

搜索到私有库，安装失败

查看一下，在使用的 Podfile 文件中，有没有添加自己的私有库地址，需要加上的，否则安装不成功

就是这句 source ‘http://192.168.10.224:3000/iOS/iOSPod.git‘

构建 Pods 库

构建Pods私有库 or 公有库，其实没有固定的步骤，只要做全就OK了。

通过脚本构建，非常方便

命令解析：

将 podspec 文件推送到仓库，这里需要区分是推送到公共仓库还是私有仓库

ERROR:The repo iOSPod at ../../../.cocoapods/repos/iOSPod is not clean

删除 iOSPods 重新添加一次就好了

i386 问题

描述：在使用第三方库的时候，遇到了 i386 验证不通过，修改验证参数

–skip-import-validation, 跳过验证
–use-libraries，使用了静态库

文件夹不见了

在class或者asset文件夹下的文件，通过pod update 之后不见了
原因：

  1. 文件夹是空的，肯定不见了
  2. spec文件中有没有修改

想上面这个 xxxx.xcassets 文件通过pod update 之后就不会再现了

如果遇到某个模块没有的可以通过 warning 来标志

比如某个依赖的模块没有，可以通过这个进行标识，之后全局搜索就可以了

pch 公共类

不要单独写 pch 文件，需要在spec文件中指明

环境问题

  1. 因为 py脚本使用了 Python 库，所以需要安装 Python 的 requests 库
  2. 整个项目中所有的Cocopods的版本必须一致，否则会出现问题，我也不知道是什么问题
  3. 脚本遇到问题，无非是变量或者路径问题，仔细看日志

程序运行报 i386 错误：

看一下 Podfile 文件中的 use_frameworks! 是否注释掉了

执行脚本验证通过但是push的时候失败

删除缓存路径下的 external 文件夹下响应的模块内容

单元测试

在执行单元测试的时候发现找不到对应的测试文件，导入了之后，提示 not find file xxx.h
两步解决：

  • 修改podfile 文件,测试的target一定要包含进来
  • 测试的 Other Linker flag 和主工程一致

目前来看，这两步设置了之后就可以愉快的测试了。

–verbose

在进行 lint 和 push 的时候添加 –verbose 参数，查看错误信息，一般是报找不到某个类的错误，这个时候需要通过 <> 引入，如果还是验证不通过就需要删除缓存中external中的响应模块信息了

调试：

在提交私有pod库的时候，并不总是会成功，如果有警告我们可以忽略，但是一旦有error，就不能提交成功了

  1. 这里 –verbose 是指的查看错误详细信息，可以通过这里的错误信息找到问题所在
    –allow-warnings 是允许警告的存在，这里因为有些库的注释警告没办法避免了。

  2. 在库引入依赖库中的文件的时候，最好使用这种方式，否则有些文件可能找不到

  3. 在引入自己的私有库的时候最好加上最新的版本，有时候加载进来的不一定是最新的库，这个根据本地的索引表有可能不同。
