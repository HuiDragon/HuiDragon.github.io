SQL 语言介绍：

SQL，全称数据化结构查询语言，常见的SQL语句类型分为：数据定义语言（DDL）、事务处理语言（TPL）、数据控制语言（DCL）和数据操作语言（DML）。对于开发而言，着重使用的是DML，DML语言包含了Select、Insert、Update和Delete等操作。正确使用SQL语句，有利于：增加数据库处理效率，减少应用相应的时间；减少数据库服务的负载，增加服务器的稳定性；减少服务器间通讯的网络流量。

发表于 2014-08-09 | 分类于 MySQL

约束，是否可以为空的维度

  1. 约束保证数据的完整性和一致性；
  2. 约束分为表级约束和列级约束；
  3. 约束类型分为：
    • NOT NULL(非空约束)
    • PRIMARY KEY(主键约束)
    • UNIQUE KEY(唯一约束)
    • DEFAULT(默认约束)
    • FOREIGN KEY(外键约束)

FOREIGN KEY外键约束

保持数据一致性，完整性；实现一对一或者一对多关系。

外键约束的要求：

  1. 父表和子表必须使用相同的存储引擎，而且禁止使用临时表；
  2. 数据表的存储引擎智能为 InnDB；
  3. 外键列和参照列必须具有相似的数据类型。其中数字的长度或是否有符号为必须相同；而字符的长度则可以不同；
  4. 外键列和参照列必须创建索引。如果外键列不存在索引的话 MySQL 讲自动创建索引。

外键约束的参照操作

  1. CASCADE：从父表删除或者能够更行某行，子表自动更新或者删除某一行；
  2. SET NULL: 从父表删除或者更新行，并设置子表中的外键列为NULL，必须保证子表没有指定 NOT NULL；
  3. RESTRICT: 拒绝父表的删除或者更新操作；
  4. NO ACTION：标准的SQL的关键字，功能与 RESTRICT 相同。

操作数目的维度 —— 表级约束和列级约束

  • 对一个数据列设置的约束，称为列级约束，列级约束既可以在定义列的时候声明，也可以在定义列之后生命；
  • 对多个数据列设置约束，称为表级约束，表级约束只能在定义列的时候生命。

数据类型

数据类型是指的列、存储过程参数、表达式和局部变量的数据特征，它决定了数据的存储格式，代表了不同的信息类型。

整型：

  1. TININT， 1 个字节
  2. SMALLINT，2 个字节
  3. MEDIUMINT， 3 个字节
  4. INT， 4 个字节
  5. BIGINT， 8 个字节

数据类型的选取，选取最合适的数据类型，才更加有利于我们优化数据库。

浮点型

  1. FLOAT[(M,D)]
  2. DOUBLE[(M,D)]

日期时间类型

  1. YEAR
  2. TIME
  3. DATE
  4. DATETIME
  5. TIMETAMP

字符型

  1. CHAR(M), M 个字节，定长类型， 0 \<= M \<= 255
  2. VARCHAR(M)
  3. TINYTEXT
  4. TEXT
  5. MEDIUMINT，
  6. LONGTEXT
  7. ENUM(‘value1’,’value2’,…)
  8. SET(‘value1’,’value2’,…)

空值与非空值

  • NULL， 字段值可以为空；
  • NOT NULL， 字段值不可以为空；

AUTO_INCREMENT 自动编号

  • 自动编号，且必须与主键组合使用；
  • 默认情况下，起始值为1，每次的增量为1；

PRIMARY KEY

  • 主键约束
  • 每张数据表只能存在一个主键
  • 主键保证记录的唯一性
  • 主键自动为 NOT NULL

创建一个数据表 tb3, 设置 id 为主键，并且插入三条数据，查看 id 自动增长。

注意 AUTO_INCREMENT 一定需要和PRIMAR KEY 一起使用，但是 PRIMARY KEY 可以单独使用。并且主键不能重复。

UNIQUE KEY

  • 唯一约束
  • 唯一约束可以保证记录的唯一性
  • 唯一约束的字段可以为空值（NULL）
  • 每张数据表可以存在多个唯一约束

默认约束

  • 默认值
  • 当插入就时，如果没有明确为字段赋值，则自动赋予默认值。
