Create Unit Test Target & Class

Target 在项目创建的时候如果勾选上Unit Test 那么就会自动生成一个测试target，如果没有勾选按照线面的方法再去创建一个就好了。

打开Xcode，在导航栏上选中第五个按钮，点击左下角 + 按钮，如果已经存在 Unit Test Target，那么可以选择创建一个Unit Test Class，如果没有这个Target那么需要首先创建Target。这里创建一个Class，集成自 XCTestCase ，注意所有的测试类都统一继承自这个类。类名为 CalculatorTests

- setUp 测试代码执行之前都会执行的一个方法，可以在里面做一些初始化工作，相当于开发类的 init 方法；
- tearDown 测试代码执行结束之后都会执行的一个方法，可以在里面做一些销毁工作，相当于开放类的的案例dealloc方法
- testExample 顾名思义，一个示例方法，你写的测试方法要和它的格式一致及-testxxx
- testPerformanceExample 这个是性能测试方法；
CalculatorTests : XCTestCase 自定义的测试类继承自 XCTestCase

测试 Calculator

  1. 引入 Calculator 头文件；
  2. 声明Calculator属性；
  3. 在-setup方法中创建这个对象；
  4. 添加-testCaculatorNotNil方法，注意方法必须是testXXXX类型，这是规定；

  6. 执行测试，在方法的前面有一个小的菱形，点击就可以执行测试了，如果没有请按 command + s保存一下；
  7. 可以将self.calculator = [[Calculator alloc] init];注释掉在执行测试看看结果。

对单元测试认知

单元测试是TDD的一部分，对开发人员的编程能力要求也比较高，你不仅仅需要会编程更需要知道你为什么这么写，这么写到底有哪些逻辑要考虑。对于刚开始写单元测试的人来说，我觉得就是一个字“写”，写的多了慢慢的自然就会理解里面的东西了。首先，需要理解苹果给的测试的API，这是Unit Test的工具类。其次，需要练习使用这些API，写一写简单的测试类和方法。最后，需要通过测试的结果去优化我们写得代码。当然，最后要去维护我们的测试代码的，这是必需的，否则你的测试可能进行不下去。

Requests 库详细介绍

七个主要方法

requests.request(): 构造一个请求，支撑以下各个方法的基础方法；
requests.get(): 获取HTML网页的主要方法，对应于HTTP的GET方法；
requests.head():获取HTML网页头信息的方法，对应HTTP的HEAD
requests.post():向HTML网页提交POST请求的方法，对应HTTP的POST
requests.put():向HTTML网页提交PUT请求的方法，对应于HTTP的PUT
requests.patch(): 向HTML网页提交局部修改请求，对应于HTTP的PATCH
requests.delete(): 向HTML页面提交删除请求，对应于HTTP的DELETE

Requests 库的get()方法

构造一个向服务器请求资源的Request对象，返回一个包含服务器资源的Response对象。

get() 方法的完整格式：

url: 获取页面的url链接；
params： url中的额外参数，字典或字节流格式，可选；
**kwargs：12个空值访问的参数。

get 方法的定义：

Response 对象

r.status_code 为 200 表示请求成功
type(r),查看 r 的类型，为Response类
r.headers 查看头部信息

Response 对象的5个属性

  1. r.status_code: HTTP 请求返回的状态， 200 表示连接成功， 404 表示失败；
  2. r.text: HTTP响应内容的字符串形式，即 url对应的页面内容
  3. r.encoding 从HTTP header中猜测的响应内容编码方式
  4. r.apparent_encoding: 从内容中分析出的响应内容编码方式（备选编码方式）
  5. r.content: HTTP 响应内容的二进制形式

Requests 库的异常

  1. requests.ConnectionError: 网络连接错误，如DNS查询失败、拒绝连接；
  2. requests.HTTPError： HTTP 错误异常，出现在HTTP协议称的错误；
  3. requests.URLRequired： URL 确实异常；
  4. requests.TooManyRedirects： 超过最单重定向次数，产生重定向异常；
  5. requests.ConnectTimeout：连接服务器超时异常；
  6. requests.Timeout: 请求URL超时，产生异常。

如何获取 HTTP 异常类型
r.raise_for_status(): 如果请求code不是200，这个函数就会产生一个requests.HTTPError
