MySQL 基本使用之数据类型和操作数据表

发表于 2014-08-08 | 分类于 MySQL

数据类型

数据类型是指的列、存储过程参数、表达式和局部变量的数据特征，它决定了数据的存储格式，代表了不同的信息类型。

整型：

  1. TININT， 1 个字节
  2. SMALLINT，2 个字节
  3. MEDIUMINT， 3 个字节
  4. INT， 4 个字节
  5. BIGINT， 8 个字节

数据类型的选取，选取最合适的数据类型，才更加有利于我们优化数据库。

浮点型

  1. FLOAT[(M,D)]
  2. DOUBLE[(M,D)]

日期时间类型

  1. YEAR
  2. TIME
  3. DATE
  4. DATETIME
  5. TIMETAMP

字符型

  1. CHAR(M), M 个字节，定长类型， 0 \<= M \<= 255
  2. VARCHAR(M)
  3. TINYTEXT
  4. TEXT
  5. MEDIUMINT，
  6. LONGTEXT
  7. ENUM(‘value1’,’value2’,…)
  8. SET(‘value1’,’value2’,…)

空值与非空值

  • NULL， 字段值可以为空；
  • NOT NULL， 字段值不可以为空；

创建一个表 tb2，设置两个字段 username，禁止为空，age，可以为空

AUTO_INCREMENT 自动编号

  • 自动编号，且必须与主键组合使用；
  • 默认情况下，起始值为1，每次的增量为1；

PRIMARY KEY

  • 主键约束
  • 每张数据表只能存在一个主键
  • 主键保证记录的唯一性
  • 主键自动为 NOT NULL

创建一个数据表 tb3, 设置 id 为主键，并且插入三条数据，查看 id 自动增长。

注意 AUTO_INCREMENT 一定需要和PRIMAR KEY 一起使用，但是 PRIMARY KEY 可以单独使用。并且主键不能重复。

UNIQUE KEY

  • 唯一约束
  • 唯一约束可以保证记录的唯一性
  • 唯一约束的字段可以为空值（NULL）
  • 每张数据表可以存在多个唯一约束

默认约束

  • 默认值
  • 当插入就时，如果没有明确为字段赋值，则自动赋予默认值。

  2. 数据表操作：插入记录和查找记录
  3. 记录操作：创建数据表和约束的使用；

MySQL 常用命令

  1. 显示当前服务器版本： SELECT VERSION();

  2. 显示当前日期时间： SELECT NOW();

  3. 显示当前用户： SELECT USER();

MySQL 语句规范

  • 关键字与函数名称全部使用大写；
  • 数据库名称、表名称和字段名称全部使用小写；
  • SQL 语句必须以;结尾。
